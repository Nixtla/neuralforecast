## <kbd>class</kbd> `BiTCN`
BiTCN 

Bidirectional Temporal Convolutional Network (BiTCN) is a forecasting architecture based on two temporal convolutional networks (TCNs). The first network ('forward') encodes future covariates of the time series, whereas the second network ('backward') encodes past observations and covariates. This is a univariate model. 



**Args:**
 
 - <b>`h`</b> (int):  forecast horizon. 
 - <b>`input_size`</b> (int):  considered autorregresive inputs (lags), y=[1,2,3,4] input_size=2 -> lags=[1,2]. 
 - <b>`hidden_size`</b> (int):  units for the TCN's hidden state size. Default: 16. 
 - <b>`dropout`</b> (float):  dropout rate used for the dropout layers throughout the architecture. Default: 0.1. 
 - <b>`futr_exog_list`</b> (list):  future exogenous columns. 
 - <b>`hist_exog_list`</b> (list):  historic exogenous columns. 
 - <b>`stat_exog_list`</b> (list):  static exogenous columns. 
 - <b>`exclude_insample_y`</b> (bool):  the model skips the autoregressive features y[t-input_size:t] if True. Default: False. 
 - <b>`loss`</b> (nn.Module):  PyTorch module, instantiated train loss class from [losses collection](https://nixtlaverse.nixtla.io/neuralforecast/losses.pytorch). 
 - <b>`valid_loss`</b> (nn.Module):  PyTorch module, instantiated valid loss class from [losses collection](https://nixtlaverse.nixtla.io/neuralforecast/losses.pytorch). 
 - <b>`max_steps`</b> (int):  maximum number of training steps. Default: 1000. 
 - <b>`learning_rate`</b> (float):  Learning rate between (0, 1). Default: 1e-3. 
 - <b>`num_lr_decays`</b> (int):  Number of learning rate decays, evenly distributed across max_steps. Default: -1. 
 - <b>`early_stop_patience_steps`</b> (int):  Number of validation iterations before early stopping. Default: -1. 
 - <b>`val_check_steps`</b> (int):  Number of training steps between every validation loss check. Default: 100. 
 - <b>`batch_size`</b> (int):  number of different series in each batch. Default: 32. 
 - <b>`valid_batch_size`</b> (int):  number of different series in each validation and test batch, if None uses batch_size. Default: None. 
 - <b>`windows_batch_size`</b> (int):  number of windows to sample in each training batch, default uses all. Default: 1024. 
 - <b>`inference_windows_batch_size`</b> (int):  number of windows to sample in each inference batch, -1 uses all. Default: 1024. 
 - <b>`start_padding_enabled`</b> (bool):  if True, the model will pad the time series with zeros at the beginning, by input size. Default: False. 
 - <b>`training_data_availability_threshold`</b> (Union[float, List[float]]):  minimum fraction of valid data points required for training windows. Single float applies to both insample and outsample; list of two floats specifies [insample_fraction, outsample_fraction]. Default 0.0 allows windows with only 1 valid data point (current behavior). Default: 0.0. 
 - <b>`step_size`</b> (int):  step size between each window of temporal data. Default: 1. 
 - <b>`scaler_type`</b> (str):  type of scaler for temporal inputs normalization see [temporal scalers](https://nixtlaverse.nixtla.io/neuralforecast/common.scalers). Default: 'identity'. 
 - <b>`random_seed`</b> (int):  random_seed for pytorch initializer and numpy generators. Default: 1. 
 - <b>`drop_last_loader`</b> (bool):  if True `TimeSeriesDataLoader` drops last non-full batch. Default: False. 
 - <b>`alias`</b> (str):  optional,  Custom name of the model. Default: None. 
 - <b>`optimizer`</b> (Subclass of 'torch.optim.Optimizer'):  optional, user specified optimizer instead of the default choice (Adam). 
 - <b>`optimizer_kwargs`</b> (dict):  optional, list of parameters used by the user specified `optimizer`. 
 - <b>`lr_scheduler`</b> (Subclass of 'torch.optim.lr_scheduler.LRScheduler'):  optional, user specified lr_scheduler instead of the default choice (StepLR). 
 - <b>`lr_scheduler_kwargs`</b> (dict):  optional, list of parameters used by the user specified `lr_scheduler`. 
 - <b>`dataloader_kwargs`</b> (dict):  optional, list of parameters passed into the PyTorch Lightning dataloader by the `TimeSeriesDataLoader`. 
 - <b>`**trainer_kwargs (int)`</b>:  keyword trainer arguments inherited from [PyTorch Lighning's trainer](https://pytorch-lightning.readthedocs.io/en/stable/api/pytorch_lightning.trainer.trainer.Trainer.html?highlight=trainer). 

References: 
    - [Olivier Sprangers, Sebastian Schelter, Maarten de Rijke (2023). Parameter-Efficient Deep Probabilistic Forecasting. International Journal of Forecasting 39, no. 1 (1 January 2023): 333-345.](https://doi.org/10.1016/j.ijforecast.2021.11.011)
